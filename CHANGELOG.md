# Changelog

## 2026-04-02

### 启动与首屏

- 调整启动流程，默认进入日程页周视图
- 为启动阶段增加纯白遮罩，等待周视图首屏准备完成后再显示主界面
- `MainActivity` 改为在首屏真正 ready 后再调用 `reportFullyDrawn()`

### 日程页

- 修复周视图和月视图中的多处空白、闪退和主题色不一致问题
- 修复月视图切换时 `ConstraintLayout` 被错误强转导致的崩溃
- 周视图改为数据未齐时仅显示骨架，不再先画待办再补课程
- 收紧周视图首屏 ready 条件，避免当前课表短暂为空时提前出块
- 优化周视图骨架与 overlay 的刷新逻辑，减少无效重绘

### 历史待办

- 新增历史待办页面与适配器
- 优化历史待办页背景层级，修复透明区域露出错误背景的问题
- 历史待办入口切换动画改为淡入淡出，避免内容从下往上滑入
- 历史列表关闭默认 item 动画，减少二次闪烁

### 任务与课表

- 完整备份 / 恢复覆盖待办、重复待办、标签、标签颜色、课表、课程、时间表、节次和当前激活课表
- 补充 `TaskTagColorSummary`
- 修正当前课表读取逻辑，继续使用 `SharedPreferences(app_prefs.active_schedule_id)`

### 稳定性修复

- 修复编辑待办时的时间边界和未保存状态判断
- 修复导入课表时的上下文使用风险
- 修复重复待办生成下一实例时的空值问题
- 修复多处 `Fragment` / `View` 生命周期下的渲染时机问题

### UI 与主题

- 日程页、历史页等多个页面改为跟随运行时主题色
- 优化若干页面卡片、按钮、文案层级和空状态
- 清理多处硬编码颜色和硬编码文本

### 工程与质量

- 接入更多 `ktx` 用法，清理多处 `Color.parseColor` / `SharedPreferences` 旧写法
- 删除未使用布局、动画、菜单和 drawable 资源
- 收敛 `HardcodedText`、`UnusedResources`、`Overdraw`、`UseKtx` 等主要 lint 问题
- 保持 `./gradlew.bat :app:compileDebugKotlin` 和 `./gradlew.bat :app:lintDebug` 可通过
