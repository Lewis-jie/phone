# Changelog

## 2026-04-04

### 提醒与通知

- 在 `MyApplication` 启动时初始化通知渠道
- `MainActivity` 在 Android 13+ 主动申请 `POST_NOTIFICATIONS`
- `TaskRepository` 保存、更新和删除待办时同步调度提醒
- `ReminderScheduler` 改为优先使用 `AlarmManager.setAlarmClock(...)` 注册精确提醒
- 无精确闹钟权限时退回 `setAndAllowWhileIdle(...)`
- `BootReceiver` 在开机后重新读取全部任务并恢复未来提醒
- `ReminderReceiver` 与通知派发链路补齐日志，便于定位“未触发”或“未展示”
- 新增 `ReminderNotificationDispatcher`，统一处理通知渠道、通知开关、异常保护和投递去重
- 新增最近错过提醒的 catch-up 逻辑，应用恢复可用时会补发窗口内的提醒
- 新增电池优化设置跳转和权限引导
- 新增专用通知小图标 `ic_notification_reminder`

### 编辑页切换动画

- 统一待办编辑页、课表设置页、课程总览页、课程编辑页、时间表编辑页和导入页的切换方式
- 页面进入改为新页面整体上滑，旧页面在下层保持可见
- 页面返回改为当前页面整体下滑退出
- 使用 `fade_stay` 维持底层页面，避免动画过程中背景露白或双页同时漂移
- 修复待办编辑页“更新”按钮文字丢失问题

### 待办与日程体验

- 待办完成后新增 `Snackbar` 反馈
- 普通任务支持直接撤销完成状态
- 重复任务完成后自动生成下一次实例，并支持撤销时回收刚生成的实例
- `TaskViewModel` 在初始化以及保存重复任务后同步补齐到“今日结束”为止的实例
- 日视图、周视图、月视图新增“今天”快捷返回按钮
- 三种日程视图新增上下文提示文案
- 周视图增强对今天列的高亮

### 文档

- 重写 `README.md`，补充 GitHub 可读的项目说明、运行方式和提醒注意事项
- 更新 `CHANGELOG.md`，补齐提醒链路和动画修复记录
- 重写 `docs/project-guide.md`，补充维护者视角的架构说明和关键文件索引
